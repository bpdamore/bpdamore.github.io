Production

When I first started working at Stay Golden, most order processing steps were done manually. Queuing orders, calculating roasts, and even logging roasts were done by hand.

As we've grown, our processes have expanded to the point of it taking 2 hours to just start a roast day. I decided to automate a few things and to help the team be more efficient.

MistoBox Trade Coffee Wholesale Roast Monitor

MistoBox

Skills: Web Scraping, Browser Automation, API calls
Tools Used: Python, Pandas, Splinter, BeautifulSoup, Google Sheets API

MistoBox is a coffee subscription service that has a relatively simple order portal. Instead of copying each order into our Roast Sheet, I created a "One Click" script that does everything.

  • First, Splinter opens an automated browser to the Log-in page, fills in the Username and Password, and hits enter.
  • Second, Beautiful Soup pulls the table with the orders for the day.
  • Third, the python script loads in a Json file of current SKUs, matches the coffee order to a sku, and Pandas totals up each coffee's order.
  • Connecting with Google's API, each coffee is added to the roast sheet for the day.
  • Finally, Splinter selects all open orders and prints them out.

    • Trade

    Skills: CSV Manipulation, File Manipulation, User Input, API calls
    Tools Used: Python, Pandas, EasyGUI, Google Sheets API

    Trade Coffee is a larger coffee subscription, but their order portal requires more user interaction to accept orders. So the script I wrote automates the processing of orders, but not the accepting or printing.

  • After double-clicking the script, a pop up will ask what batch number the order was. Using that number, the script searches the Downloads folder for the downloaded csv.
  • The csv is then loaded into a Pandas Dataframe, alongside a Json file containing our product SKUs.
  • The dataframe is reformatted to include the product SKUs, and it counts up the total number for each product.
  • Finally the script connects to our Roasting Google Sheet via the Sheets API to queue the orders for roasting and packaging.

    • Wholesale

    Skills: Adaptable HTML/CSS, API calls, Google Sheet Iteration
    Tools Used: Python, Pandas, Google Sheets API, Shopify API, HTML, CSS

    At Stay Golden our wholesale customers can either order from our Shopify page and pay immediately, or if they need to pay with Net30 terms, they can order through an order form we made that places their responses into our Google Sheet. On top of that, if a customer wants a custom product AND Net30 terms, we have a separate order form that goes into a separate sheet. With orders coming from 7 different sheets and Shopify, processing orders can take some time.

    My script pulls wholesale orders from everywhere we take orders, queues them up for production, and then formats them into a Packing List that opens in a browser tab. The only human interaction is to run it.

    Roast Monitor

    Skills: Regular Expressions, File Manipulation, API calls
    Tools Used: Python, Regex, Google Sheets API

    After hooking up our Coffee Roaster to a computer for Roast Monitoring, we fell into a habit of saving the roast files with the default naming schemes. After about 2,000 roasts I realized that the naming convention didn't show the weight of the batch, therefore making roast comparisons almost impossible.

    I decided to make a script that would monitor our Roast Sheet folder, open a file that is placed there, use Regular Expressions to find the batch weight in the metadata, and then to rename the file and move it to a new folder.

    The script aides in daily production by taking that weight, going to our roast queue on Google Sheets, and subtracting from the total needed poundage for the week. At all times our needed poundage is up to date, and automated.
